# 🤖 Бот-консультант 24/7 — отвечает клиентам и собирает заявки

Telegram-бот, который круглосуточно отвечает на вопросы клиентов **строго по базе
знаний бизнеса** и принимает заявки прямо в чат. На примере барбершопа «Бритва».

![демо](assets/demo.gif)

---

## Проблема

Клиенты пишут в Telegram одни и те же вопросы — «сколько стоит стрижка?»,
«работаете в воскресенье?», «где вы находитесь?» — а отвечать некому, особенно
вечером и в выходные. Человек не дождался ответа за 5 минут — ушёл к конкуренту.

## Решение

Бот-консультант на **aiogram + GigaChat**, который:

- отвечает на вопросы **по базе знаний** (услуги, цены, адрес, график) —
  человеческим языком, а не сухими шаблонами;
- **не выдумывает**: если ответа в базе нет, честно говорит «уточню у менеджера»,
  предлагает оставить заявку, а сам вопрос пересылает владельцу в Telegram;
- по кнопке **«Записаться»** собирает имя, контакт и удобное время (FSM-диалог) и
  отправляет готовую заявку владельцу + пишет её в `leads.xlsx`;
- работает **локально на long polling** — без VPS, вебхуков и белого IP.

**Главная фишка:** все ответы рулятся одним текстовым файлом
[`data/knowledge.md`](data/knowledge.md). Поменял файл — перенастроил бота под
любой бизнес за 10 минут, без программиста.

## Результат

- Клиент получает ответ мгновенно, в любое время суток.
- Ноль выдуманных цен и услуг — бот отвечает только по базе (анти-галлюцинация
  через маркер `NO_ANSWER`).
- Каждая заявка зафиксирована в двух местах: уведомление владельцу + строка в Excel.

---

> ⚠️ **О модели и статусе проекта.** Это **демонстрационный (тестовый) вариант** —
> его задача показать, что проект рабочий: связка aiogram + GigaChat, ответы по
> базе знаний, анти-галлюцинация и сбор заявок. В демо используется **бесплатная,
> сравнительно слабая модель GigaChat (тариф для физлиц)**. Она может **ошибаться,
> галлюцинировать и не всегда строго следовать инструкциям** из промпта (например,
> иногда отвечать вместо честного «уточню у менеджера»). На более сильной модели
> (GigaChat Pro/Max или другой LLM) качество ответов и дисциплина заметно выше — для
> боевого использования достаточно поменять модель/ключ, код менять не нужно.

## Как это работает

```
        [Клиент в Telegram]
               │  текст / кнопки
               ▼
   ┌───────────────────────────┐
   │   aiogram bot (polling)   │
   └───────────────────────────┘
        │                    │
   свободный вопрос     кнопка «Записаться»
        │                    │
        ▼                    ▼
  ┌──────────────┐    ┌──────────────────────┐
  │   GigaChat   │    │  FSM: имя → контакт  │
  │ + база знаний│    │   → время → подтвер. │
  └──────┬───────┘    └──────────┬───────────┘
         │ ответ ИЛИ NO_ANSWER    │
         ├──── NO_ANSWER ────────►│ (вопрос → владельцу)
         ▼                        ▼
  [ответ клиенту]      ┌────────────────────────────┐
                       │ leads.xlsx + чат владельца │
                       └────────────────────────────┘
```

Анти-галлюцинация: в системный промпт кладётся инструкция + вся база знаний.
Если в базе нет ответа — модель возвращает ровно `NO_ANSWER`, код ловит маркер и
уводит клиента к менеджеру вместо того, чтобы фантазировать.

## Стек

- **Python 3.12 + [aiogram](https://docs.aiogram.dev) 3.x** — бот, меню, FSM заявки.
- **[GigaChat](https://github.com/ai-forever/gigachat) (официальный SDK)** —
  генерация ответов по базе знаний (бесплатный тариф для физлиц).
- **openpyxl** — заявки в `leads.xlsx`.
- **long polling** — запуск локально.
- **pytest** — юнит-тесты на логику (19 тестов).

## Структура

```
ai-bot/
├── bot.py                  # точка входа, polling, регистрация роутеров
├── handlers/
│   ├── start.py            # /start, меню, «Контакты»
│   ├── qa.py               # вопрос → GigaChat → ответ / эскалация
│   ├── lead.py             # FSM сбора заявки
│   └── keyboards.py        # клавиатуры и кнопки
├── services/
│   ├── gigachat.py         # обёртка SDK + детект NO_ANSWER + заглушка без ключа
│   ├── knowledge.py        # чтение базы знаний с кэшем
│   ├── leads.py            # дозапись заявок в xlsx
│   └── notify.py           # уведомления владельцу
├── core/config.py          # пути и секреты (один файл)
├── prompts/system_prompt.txt
├── data/knowledge.md       # БАЗА ЗНАНИЙ (правит клиент)
├── tests/                  # pytest
└── secrets.env.example     # шаблон секретов
```

## Запуск

```bash
# 1. Окружение
python -m venv .venv
.venv\Scripts\activate        # Windows (Linux/Mac: source .venv/bin/activate)
pip install -r requirements.txt

# 2. Секреты
cp secrets.env.example secrets/ai-bot.env
#   впиши TELEGRAM_BOT_TOKEN (у @BotFather), OWNER_CHAT_ID и GIGACHAT_CREDENTIALS

# 3. Старт
python bot.py
```

**Без ключа GigaChat** бот всё равно запустится: меню и сбор заявок работают
полностью, а на вопросы бот вежливо отвечает «уточню у менеджера» и зовёт оставить
заявку. Так демо собирается даже до получения ключа.

Где взять `GIGACHAT_CREDENTIALS` — в
[личном кабинете GigaChat API](https://developers.sber.ru/portal/products/gigachat-api)
(бесплатный scope `GIGACHAT_API_PERS`).

## Тесты

```bash
pytest
```

Покрыты: чтение базы знаний и перечитка по изменению файла, сборка промпта и
детект `NO_ANSWER`, режим заглушки без ключа, запись заявок в xlsx, переходы FSM.
Сеть, GigaChat и Telegram в тестах не дёргаются (мокаются).

## Перенастроить под свой бизнес

Открой [`data/knowledge.md`](data/knowledge.md), замени услуги/цены/адрес/график на
свои — и бот готов. Никакого кода трогать не нужно.
